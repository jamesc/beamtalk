## Summary

Completes ADR 0017 Phase 0 wire check — validates the full browser → eval → Transcript → browser round-trip over WebSocket.

**Linear issue:** https://linear.app/beamtalk/issue/BT-686

## Changes

### Erlang Runtime
- **beamtalk_transcript_stream.erl** — Added `subscribe/1` and `unsubscribe/1` module-level exports wrapping the legacy cast protocol
- **beamtalk_ws_handler.erl** — Subscribe to Transcript after auth, forward `{transcript_output, Text}` as `{"push": "transcript", "text": "..."}` push messages, unsubscribe on disconnect
- **beamtalk_repl_server.erl** — Added `cowboy_static` routes to serve `priv/index.html` at `/` and static files at `/static/[...]`

### Browser
- **priv/index.html** — Minimal workspace page (~175 lines) with eval textarea, Transcript div, cookie auth, Ctrl+Enter shortcut

### CLI Protocol
- **protocol.rs** — Filter push messages in `read_response`, `send_request`, and `read_response_line` to prevent interference with request/response flow
- **e2e.rs** — Filter push messages in E2E test harness `read_text`

### Cross-Platform Process Management (ADR 0027)
- **Cargo.toml + beamtalk-cli/Cargo.toml** — Added `sysinfo` crate for cross-platform process operations
- **process.rs** — Replaced `ps` command with `sysinfo` for PID discovery (`find_beam_pid_by_node`)
- **process.rs** — Replaced `kill -9` / `taskkill` with `Process::kill()` for force-kill (`force_kill_process`)
- **workspace/mod.rs** — Enabled 3 previously Unix-only integration tests on Windows using `sysinfo`
- **repl/mod.rs** — Enabled `beam_child_guard_kills_process_on_drop` test on Windows
- **repl_startup.rs** — Fixed clippy documentation warnings (missing backticks)

**Benefits:**
- ✅ Fixed Windows "process ID unknown" error when stopping workspaces
- ✅ Eliminated shell command dependencies (ps, kill, taskkill)
- ✅ Enabled 4 previously Unix-only tests on Windows
- ✅ More reliable process management across all platforms

## Testing
- All CI checks pass (build, clippy, fmt, dialyzer, 1879 Erlang tests, 1549 stdlib tests, 9 integration tests, 11 MCP tests, E2E tests)
- Manual test: open browser to `http://localhost:<port>/`, type `Transcript show: 'hello'`, see output in Transcript div
